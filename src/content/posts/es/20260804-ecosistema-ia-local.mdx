El post anterior cubrió por qué importan los LLM locales y cómo ejecutar tu primer modelo con Ollama. Pero un modelo corriendo en un terminal es solo el punto de partida. Para reemplazar la IA en la nube en tu flujo de trabajo diario, necesitas una capa de API, una interfaz de chat e integración con las herramientas que ya usas.

El ecosistema de IA local ha madurado lo suficiente para proporcionar las tres cosas. Este post recorre el stack: LocalAI como gateway de API, Open WebUI como interfaz y las herramientas que conectan la inferencia local con el trabajo real.

## LocalAI: el gateway compatible con OpenAI

LocalAI es un reemplazo directo de la API de OpenAI. Sirve modelos de múltiples backends — Ollama, llama.cpp, vLLM y otros — a través de un único endpoint de API que sigue la especificación de OpenAI.

```bash
# Ejecutar LocalAI con backend Ollama
docker run -p 8080:8080 \
  -v LocalAI/models:/models \
  localai/localai:latest

# Listar modelos disponibles
curl http://localhost:8080/v1/models

# Chat completion (compatible con OpenAI)
curl http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "llama3.1:8b",
    "messages": [{"role": "user", "content": "Escribe un script bash para monitorear el uso de disco."}]
  }'
```

El valor de LocalAI es la abstracción. Cualquier aplicación construida para la API de OpenAI — ChatBox, LibreChat, cientos de scripts personalizados — funciona contra LocalAI sin cambios de código. Apuntas la URL base a tu instancia de LocalAI, proporcionas cualquier cadena como clave de API, y la aplicación no nota la diferencia.

LocalAI también soporta galerías de modelos. Puedes descargar modelos directamente a través de la API en vez de gestionar archivos GGUF manualmente:

```bash
# Instalar un modelo desde la galería
curl http://localhost:8080/models/apply \
  -H "Content-Type: application/json" \
  -d '{"id": "huggingface://TheBloke/Mistral-7B-v0.1-GGUF/mistral-7b-v0.1.Q4_K_M.gguf"}'
```

Para equipos ejecutando múltiples modelos, LocalAI maneja el cambio de modelos a través de la API. Una sola instancia puede servir un modelo pequeño para tareas rápidas y uno más grande para razonamiento complejo, seleccionado dinámicamente por la aplicación que lo llama.

## Open WebUI: la interfaz en navegador

Open WebUI proporciona una interfaz tipo ChatGPT que corre completamente en tu navegador contra tu backend local. Soporta Ollama de forma nativa y puede conectarse a cualquier API compatible con OpenAI.

```bash
# Instalar Open WebUI con backend Ollama
docker run -d -p 3000:8080 \
  -e OLLAMA_BASE_URL=http://host.docker.internal:11434 \
  --name open-webui \
  ghcr.io/open-webui/open-webui:main
```

Una vez en funcionamiento, abre `http://localhost:3000` y obtienes una interfaz de chat con todas las funciones: historial de conversaciones, prompts del sistema, cambio de modelos, renderizado de markdown, resaltado de código y soporte de generación de imágenes.

Open WebUI va más allá del chat básico. Soporta:

- **RAG (Retrieval-Augmented Generation)** — Sube documentos y la interfaz los fragmenta, los incrusta y recupera contexto relevante para cada consulta. No requiere base de vectores externa.
- **Perfiles de modelo personalizados** — Configura diferentes prompts del sistema, configuraciones de temperatura y selección de modelos por tipo de conversación.
- **Soporte multi-usuario** — Si lo ejecutas en un servidor casero, familiares o compañeros de equipo pueden tener sus propias cuentas con historiales de conversación separados.
- **Integración con búsqueda web** — Conecta una API de búsqueda y el modelo puede fundamentar sus respuestas en información actual sin fine-tuning.

La función RAG es particularmente relevante para la soberanía. Puedes subir documentos sensibles — contratos, documentación interna, papers de investigación — y consultarlos sin que ningún dato salga de tu red. Las incrustaciones se computan localmente, se almacenan localmente y se recuperan localmente.

## LM Studio: la alternativa de escritorio

LM Studio es una aplicación de escritorio que combina gestión de modelos, una interfaz de chat y un servidor local en una sola descarga. Está dirigida a usuarios que quieren una experiencia gráfica sin Docker ni herramientas de línea de comandos.

La interfaz te permite navegar y descargar modelos de Hugging Face directamente, ajustar cuantización y configuraciones de contexto con deslizadores, y arrancar un servidor local con un clic. El servidor expone la misma API compatible con OpenAI que Ollama y LocalAI.

LM Studio no es open source, lo cual importa si tu definición de soberanía incluye las herramientas mismas. Es una aplicación de código cerrado que ejecuta modelos de código abierto. Para muchos usuarios, esto es un intercambio aceptable. Para otros, lo descalifica.

## Continue.dev: IA en tu editor

Continue.dev es un asistente de código AI de código abierto que se conecta a modelos locales. Corre como extensión de VS Code o JetBrains y soporta Ollama, LM Studio y cualquier endpoint compatible con OpenAI.

```json
{
  "models": [
    {
      "title": "Llama Local",
      "provider": "ollama",
      "model": "llama3.1:8b"
    }
  ],
  "tabAutocompleteModel": {
    "title": "Autocompletado de Código",
    "provider": "ollama",
    "model": "deepseek-coder:6.7b"
  }
}
```

Configúralo para usar un modelo más grande para chat y uno más pequeño y rápido para autocompletado de pestañas. El autocompletado corre en segundo plano, sugiriendo completados mientras escribes, sin enviar tu código a ningún servicio externo.

Aquí es donde la IA local se vuelve genuinamente útil para trabajo de desarrollo. El modelo entiende el contexto de tu base de código, sugiere completados que encajan con tus patrones y responde preguntas sobre el código que estás viendo — todo sin que tu código salga jamás de tu máquina.

## Local vs. Nube: la comparación honesta

La IA local no es estrictamente mejor que la IA en la nube. Los intercambios son reales:

- **Calidad** — GPT-4o y Claude 3.5 Sonnet aún superan a modelos locales de 7B-70B en razonamiento complejo, tareas de contexto largo e instrucciones matizadas. La brecha se está reduciendo, pero existe.
- **Velocidad** — Una GPU de consumo ejecutando un modelo de 7B genera 30-50 tokens por segundo. GPT-4o genera más rápido en la API, pero pagas por token y esperas latencia de red.
- **Coste** — Un setup local cuesta hardware por adelantado (una GPU de $300 se paga sola en meses de ahorro en API). La IA en la nube no cuesta nada por adelantado pero cobra por solicitud indefinidamente.
- **Privacidad** — Local gana incondicionalmente. Ningún dato sale de tu máquina. Ningún término de servicio regula tu uso. Ningún proveedor puede analizar tus prompts.
- **Fiabilidad** — La inferencia local funciona offline, sin límites de velocidad, sin interrupciones, sin avisos de descontinuación. Es la IA más fiable que puedes tener, porque no depende de nada más que de tu propio hardware.

La respuesta práctica no es una u otra. Usa modelos locales para tareas diarias, datos sensibles y cualquier cosa que quieras mantener privada. Usa modelos en la nube para la tarea compleja ocasional que excede tu hardware local. La soberanía viene de tener la elección.

## Construyendo tu stack

El ecosistema de IA local ya no es un concepto de prueba. Es una alternativa funcional a las APIs en la nube para la mayoría de casos de uso diarios. Los componentes están maduros, las herramientas están conectadas y los modelos mejoran a un ritmo que reduce la brecha cada trimestre.

Empieza con Ollama para el backend del modelo. Añade Open WebUI para la interfaz. Conecta Continue.dev para asistencia de código. El stack corre en una sola máquina, no requiere cuentas en la nube y no envía nada a ningún sitio.

Así es como se ve la soberanía sobre la IA en la práctica.
